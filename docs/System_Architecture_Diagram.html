System Architecture

EXPLAINABLE MEDICAL DIAGNOSIS MODEL — END-TO-END PIPELINE

📥
Input — NIH Chest X-ray14 Dataset
Raw medical imaging data
112,120 frontal chest X-ray images (PNG format)
14 disease labels (multi-label classification)
984 images with bounding box ground truth annotations
Dataset size: ~45 GB
DATA SOURCE
↓
⚙️
Data Preprocessing Pipeline
Image transformation & normalization
Resize images to 224×224 pixels
Convert grayscale to 3-channel RGB (repeat channels)
Normalize using ImageNet statistics (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
Train/Val/Test split: 80% / 10% / 10%
Compute class weights for handling imbalance
MODULE 1
↓
🧠
ResNet50
Deep residual network
50 layers, ~25M parameters
Pre-trained on ImageNet
Skip connections prevent vanishing gradient
Final layer: Linear(2048 → 14) + Sigmoid
Training: BCEWithLogitsLoss, Adam optimizer
MODEL A
🧠
EfficientNet-B0
Compound-scaled network
Lightweight, ~5.3M parameters
Pre-trained on ImageNet
MBConv blocks with SE modules
Final layer: Linear(1280 → 14) + Sigmoid
More memory efficient than ResNet50
MODEL B
↓
📊
Disease Classification Output
Multi-label probability scores
14 independent disease probability scores [0, 1]
Target performance: AUC-ROC ≥ 0.80 per class
Both models evaluated on same test set for fair comparison
MODULE 2 OUTPUT
↓
🔍
Grad-CAM
Gradient-weighted activation mapping
Hook into final convolutional layer
Compute gradient of class score w.r.t. feature maps
Generate 7×7 coarse heatmap
Upsample to 224×224 using bilinear interpolation
Fast: milliseconds per image
XAI METHOD 1
🔍
GradientSHAP
Shapley-based attribution
Define Gaussian noise baseline distribution
Integrate gradients along baseline → input path
Compute pixel-level SHAP values
Generate full 224×224 attribution map
Precompute and cache (slower than Grad-CAM)
XAI METHOD 2
↓
🗺️
Explanation Heatmap Generation
Visual attribution maps for each prediction
4 sets of heatmaps: (ResNet50 + Grad-CAM), (ResNet50 + SHAP), (EfficientNet + Grad-CAM), (EfficientNet + SHAP)
984 test images × 4 combinations = 3,936 total heatmaps
All maps normalized to [0, 1] range
Cached as .npy files for efficient metric computation
