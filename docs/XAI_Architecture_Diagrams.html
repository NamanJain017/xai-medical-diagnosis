Full System Pipeline
End-to-end data flow — from raw X-ray images to final explanation quality scores
5-module sequential pipeline — M1 → M2 → M3 → M4 → M5
M1 — Data Pipeline
Download NIH Chest X-ray14 dataset
Parse CSV for labels and bounding boxes
Resize images to 224×224 px
Normalize using ImageNet stats
Convert grayscale → 3-channel
80/10/10 train/val/test split
Compute class weights for imbalance
Output: Tensor DataLoaders
M2 — CNN Models
Load ResNet50 (pretrained ImageNet)
Load EfficientNet-B0 (pretrained ImageNet)
Replace final layer → Linear(n, 14)
Sigmoid activation (multi-label)
Train with BCEWithLogitsLoss
Mixed precision (AMP) training
Evaluate per-class AUC-ROC ≥ 0.80
Output: Trained model weights
M3 — XAI Engine
Hook into final conv layer (Grad-CAM)
Compute gradient-weighted activations
Generate Grad-CAM heatmaps
Define Gaussian noise baseline (SHAP)
Compute GradientSHAP attributions
Normalize all maps to [0, 1]
Cache 984 × 4 maps to disk (.npy)
Output: 4 sets of explanation maps
M4 — Evaluation Engine
Load bounding box ground truth
Threshold heatmap → binary mask
Compute IoU (Metric 1)
Remove top-K% pixels, re-infer
Measure confidence drop (Metric 2)
Add Gaussian noise, re-explain
Cosine similarity of maps (Metric 3)
Output: 3 scores × 4 combinations
M5 — Analysis & Report
Aggregate all metric scores
Build 4×3 comparison table
Plot fidelity curves (K% vs drop)
Generate heatmap overlay figures
Statistical analysis (mean ± std)
Write paper / project report
Target: IEEE CBMS / ICMLA
Output: Paper + benchmark results

XAI Engine — Grad-CAM vs GradientSHAP
Two post-hoc explanation methods applied to both models — step-by-step internal process
Side-by-side process comparison — same input, different explanation algorithms
Grad-CAM
Gradient-weighted Class Activation Mapping
1
Forward pass — run X-ray through modelGet prediction (e.g. Pneumonia = 0.87)
2
Hook into final conv layer (layer4 / last MBConv)Capture the feature map activations at this layer
3
Backpropagate gradient of class score∂(disease score) / ∂(feature maps)
4
Global average pool the gradientsEach feature map gets one weight value
5
Weighted sum of feature mapsα₁·A¹ + α₂·A² + ... → coarse heatmap (7×7)
6
Apply ReLU — keep only positive contributionsNegative values = regions that reduce prediction
7
Upsample to 224×224 (bilinear interpolation)Overlay on original X-ray image
✓ Milliseconds per image — very fast
✓ No model modification needed
✗ Resolution limited by final conv layer size (7×7)
✗ Only one class at a time
vs
GradientSHAP
SHapley Additive exPlanations (deep variant)
1
Define baseline distributionN random Gaussian noise images — the "uninformative" reference
2
Sample random baseline from distributionEach attribution calculation uses a fresh noise sample
3
Interpolate path from baseline → inputCreate a straight-line path of images between noise and real X-ray
4
Compute gradient at each interpolation point∂(output) / ∂(input) at each point on the path
5
Integrate gradients (expected value over path)Average across all interpolation points and baselines
6
Multiply by (input − baseline)Gives each pixel its final SHAP attribution value
7
Output pixel-level attribution map (224×224)Both positive (+) and negative (−) values per pixel
✓ Theoretically grounded — Shapley axioms satisfied
✓ Full 224×224 resolution output
✗ 10–20× slower than Grad-CAM
✗ Must precompute and cache results
Evaluation Metrics — The Research Contribution
Three quantitative metrics that measure whether an explanation can actually be trusted
3-metric evaluation framework — applied to all 4 model+XAI combinations
1
Localization Accuracy
IoU Score
Does the explanation point to the right anatomical region?
Threshold heatmap at top 50th percentile → binary mask
Load NIH bounding box ground truth annotation
Compute pixel-wise overlap between mask and bbox
Divide overlap by total union area
IoU = Overlap Area / Union Area
↑ High = pointing at disease region
↓ Low = highlighting wrong area
2
Explanation Fidelity
Deletion / Confidence Drop Test
Are the highlighted pixels actually responsible for the prediction?
Record original model confidence S₀
Identify top-K% most important pixels from heatmap
Replace those pixels with mean pixel value (mask)
Re-run model, record new confidence S₁
Fidelity = S₀ − S₁ (confidence drop)
↑ Large drop = pixels were genuine
↓ Small drop = explanation was misleading
3
Explanation Stability
Perturbation Consistency
Does the explanation stay consistent for nearly identical inputs?
Generate explanation E₁ for original image
Add imperceptible Gaussian noise (σ=0.02) → I′
Generate explanation E₂ for noisy image I′
Flatten both maps to vectors, compute similarity
Stability = CosSim(E₁, E₂)
↑ High = stable, reliable explanation
↓ Low = flips on tiny input changes

4-Combination Comparison Framework
2 models × 2 XAI methods = 4 combinations evaluated on 3 metrics — this is your result table
What makes each combination unique
ResNet50 + Grad-CAM
The baseline combination. Most widely studied in existing literature. Grad-CAM hooks into layer4. Coarse 7×7 heatmap. Benchmark against which all other combinations are compared.
ResNet50 + SHAP
Same architecture, different XAI. Isolates the effect of the explanation method on a fixed model. Tests if SHAP's theoretical grounding translates to better fidelity and stability scores.
EfficientNet-B0 + Grad-CAM
Same XAI, different architecture. Isolates the effect of model design on explanation quality. Tests whether a lighter model with SE modules produces better or worse localizations.
EfficientNet-B0 + SHAP
Full combination — lightweight modern architecture + theoretically grounded XAI. Expected to perform well but hypothesis is not assumed — your results will tell the real story.
